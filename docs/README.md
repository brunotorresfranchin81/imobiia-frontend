# Imobiia Frontend - TanStack Start v2

**Real TanStack Start** with React 19, TanStack Router, Supabase, Tailwind v4, and shadcn/ui.

## Quick Start

```bash
# Install dependencies (already done)
npm install

# Configure Supabase
cp .env.example .env.local
# Edit .env.local with your credentials

# Start development server
npm run dev
# http://localhost:3000

# Build for production
npm run build

# Type checking
npm run typecheck
```

## Tech Stack

- **Framework**: TanStack Start (React 19) - **Real** TanStack Start, not just Router
- **Routing**: TanStack Router (v1 API - createRootRoute/createFileRoute)
- **Queries**: TanStack Query (via TanStack Start)
- **Auth**: Supabase (Client-Side Rendering + AuthClaims)
- **UI**: shadcn/ui + Tailwind CSS v4
- **Types**: TypeScript (strict mode)
- **Build**: Vite v8
- **Styling**: Tailwind v4 (CSS-first, CSS variables for IMOBIIA theme)

## Key Features

### Authentication
- `getAuthClaims()` - Extract role, companyId from Supabase JWT
- `useAuth()` hook - Real-time auth state + onAuthStateChange
- UserRole type: 'admin' | 'gestor' | 'corretor'

### Styling
- CSS variables for IMOBIIA theme (azul-petróleo, areia)
- Dark mode support (CSS vars)
- shadcn/ui Nova preset with Neutral colors

### Routes
- File-based routing (automatic from src/routes/)
- Route tree generated by `tsr generate`
- Lazy code splitting enabled


## Important Notes

1. **CSR Only**: This is Client-Side Rendering. For SSR, use `@supabase/ssr` + TanStack Start adapters.
2. **Tailwind v4**: CSS-first config in `src/styles.css` (no `tailwind.config.js`).
3. **shadcn/ui**: Components copied to project (not npm packages), fully customizable.
4. **database.types.ts**: Auto-generated from Supabase. Regenerate when schema changes.
5. **Route Tree**: Auto-generated from file structure. Run `npm run generate-routes` after adding routes.

## Database Types

Regenerate after schema changes:

```bash
npx supabase gen types typescript --project-id YOUR_PROJECT_ID > src/lib/database.types.ts
```
